Vagrant

  • Cドライブにboxを追加する
  • Cドライブに新しいフォルダをつくる
  • boxを初期化する
  • 仮想マシンに接続する
  • 仮想マシンを設定していく (sshで接続したまま)
  • ファイアウォールを切る
  • キャッシュ対策
  • 仮想マシンを再起動する
  • 共有フォルダを作成する
  • ブラウザから見られるようにする

Cドライブにboxを追加する (CentOS7の場合) Local

Vagrant Boxes

Cドライブ直下で、$vagrant box add [ボックスの名前 パッケージ]

$vagrant box list (確認)

$vagrant box remove [ボックスの名前 パッケージ] (削除)

※この時点でCドライブには「.vagrant」というフォルダが出来ていればOK。

Cドライブに新しいフォルダをつくる Local

$mkdir [新しいディレクトリ名 ex: test] (この中に後ほど仮想マシンと同期する共有ディレクトリを設置していく)

$cd [新しいディレクトリ名 「test」] (作ったフォルダに移動)

boxを初期化する Local

$vagrant init [ボックスの名前 パッケージ] ※さっき使ったbox名

これでこのフォルダの中だけ仮想マシンと繋げられるようになった。

$vagrant up （仮想マシンを起動）

仮想マシンにssh接続する Local

$vagrant ssh

仮想マシン内の設定をしていく (以降sshで接続したまま) ssh

$sudo yum -y install httpd (Apacheをインストール)

$sudo systemctl start httpd.service (Apacheを起動)

$sudo systemctl enable httpd.service (Apacheを常時起動に設定)

ファイアウォールを切る ssh

$sudo systemctl stop firewalld

$sudo systemctl disable firewalld

キャッシュ対策 ssh

cd /etc/httpd/conf/ (httpd.confに移動)

$ sudo vi httpd.conf (編集)

<Directory /var/www/html/>
 EnableMMAP Off
 EnableSendfile Off
</Directory>

仮想マシンを再起動する Local

$ exit (ssh接続から抜ける)

$ vagrant reload (再起動)

共有フォルダを作成する (ここ以降ローカルで作業) Local

$ cd /c/[Cドライブにさっき作ったディレクトリ名 「test」] (移動)

$ vi Vargrantfile (編集する)

config.vm.synced_folder "./[共有したいディレクトリ名]", "/var/www/html" ( ./ は$ vagrant initした場所=「C:test」。ここに入れたファイルと仮想マシンの/var/www/htmlに置いたファイルが同期する）

$ vagrant reload (再起動)

centOS(linux)側の/var/www/html に置いたものと同期されるようになる。

ブラウザから見られるようにする Local

$ cd /c/[ディレクトリ名] (移動)

$ vi Vargrantfile (編集する)

config.vm.network "private_network", ip: "192.168.33.10" のコメントアウト( # )を外す

$ vagrant reload (再起動)

C:testの下層にフォルダを置いてそれらを別々に管理する方法

cd /c/test （testに移動）

mkdir ruby （ex: rubyフォルダを作る）

$ vagrant init [ボックスの名前 パッケージ] ※さっき使ったのと同じbox名で初期化

$ vi Vagrantfile (rubyフォルダ内に出来たVagrantfileを編集する)

config.vm.synced_folder "../ruby", "/var/www/html" (ひとつ下層なので../)

$ vagrant ssh以降は同じ。

192.168.33.10/ruby/ でブラウザからアクセスできる。

仮想マシン内では/var/www/html/ruby と同期している。
